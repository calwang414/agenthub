方案 A：表格视图

与插件管理页一致的数据表格布局，信息密度高、扫描效率好，适合管理大量用户。支持全选、批量禁用/启用、行内操作。

优点

  • 信息密度高，快速扫描
  • 批量操作效率高
  • 与插件管理页一致

注意

  • 移动端需横向滚动
  • 缺少用户形象感知

方案 B：卡片网格

用户以卡片形式网格排列，每个卡片承载头像、信息与操作。视觉更丰富、用户形象更突出，hover 时展示操作。适合中小规模用户管理。

优点

  • 用户形象更突出
  • 移动端友好，自然换行
  • 视觉层次丰富

注意

  • 大量用户时滚动较长
  • 批量操作不便

方案 C：分栏详情（推荐）

左侧紧凑用户列表，点击用户后右侧滑出详情面板。兼顾列表扫描与详情查看，无需弹窗即可查看完整资料和操作历史。操作在详情面板中完成。

优点

  • 列表 + 详情一屏尽览
  • 无需弹窗即可查看/编辑
  • 操作效率高，体验流畅

注意

  • 需额外状态管理
  • 移动端需叠层展示
💡 提示：方案 A（表格）与现有插件管理页面模式一致，方案 C（分栏）更现代但需要额外布局实现。你可以选择一种作为默认视图，另一种作为可选切换视图（类似插件管理页的"表格/卡片"切换）。也支持方案 A + B 双视图切换（表格默认 + 卡片可选）。
